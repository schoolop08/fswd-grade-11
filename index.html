Unit 06

Redux
Global State Management

The Problem: Props Drilling 🕳️

Imagine passing data from a grandparent component all the way down to a grandchild — through every middle component that doesn't even need it. That's props drilling, and it's a nightmare in large apps.

The Bank Analogy: You don't walk into a bank vault yourself. You submit a request (Action), a teller processes it (Reducer), and the balance updates in the system (Store). Same idea.

4 Core Concepts

Store

The single vault. All your app's state lives here. One source of truth.

Action

A plain object: { type: "INCREMENT" }. Describes what happened.

Reducer

A pure function: takes (state, action) → returns new state. Never mutates!

Dispatch

How you trigger an action: dispatch(increment())

state.counter
0
// dispatch log will appear here
counterSlice.js 
const counterSlice = createSlice({
  name: "counter",
  initialState: 0,
  reducers: {
    increment: (state) => state + 1,
    decrement: (state) => state - 1,
  }
})
export const { increment, decrement } = counterSlice.actions

// In your component:
const count = useSelector(s => s.counter) // READ
const dispatch = useDispatch()              // WRITE
dispatch(increment())

React Essentials — useState

Component.jsx 
// STATE — remembers values between renders
const [count, setCount] = useState(0)
//      ^state  ^setter      ^initial value

// Updating state triggers a re-render
setCount(count + 1)

// EFFECT — run code on mount / when deps change
useEffect(() => {
  fetch("https://api.example.com/users")
    .then(res => res.json())
    .then(data => setUsers(data))
}, []) // [] = run once on mount

Virtual DOM — How React is Fast

1️⃣
State changes

2️⃣
Virtual DOM diff
React compares old vs new virtual tree
3️⃣
Minimal update
Only changed parts update in real DOM
One-way data flow: Data flows down (parent → child via props). Events go up (child calls parent's callback).
Unit 08

npm — Node
Package Manager

npm is like the App Store for JavaScript code. Instead of writing a server from scratch, you just install Express. Instead of writing date logic, you install moment. Your project's "shopping list" of these packages is package.json.

3 Key Files

package.json

The heart. Stores name, version, scripts, dependencies. Commit this.

package-lock.json

Locks exact versions so everyone on your team gets the same code. Commit this too.

node_modules/

Where packages actually live. NEVER push to GitHub! Add to .gitignore.

Unit 09

Node.js &
Express.js

Node.js — JS outside the browser

You know JS runs in Chrome. Node.js lets you run JS on a server (or your computer). It's built on Chrome's V8 engine but has no DOM — instead, it has access to files, databases, and networks.

Key trait: Node is asynchronous and non-blocking. It can handle thousands of requests without freezing, because it doesn't wait around for slow operations.

Express.js — the thin wrapper

Node's built-in server is verbose. Express wraps it in clean, readable syntax — routing, middleware, and response helpers in a few lines.

A Working Express Server

server.js 
const express = require("express")
const app = express()

// Middleware — parses incoming JSON body
app.use(express.json())

// Routes
app.get("/", (req, res) => {
  res.send("Hello World!")
})

app.post("/login", (req, res) => {
  const { email } = req.body // from JSON body
  res.json({ message: `Welcome, ${email}` })
})

// URL param → /user/42 → req.params.id = "42"
app.get("/user/:id", (req, res) => {
  res.send(req.params.id)
})

app.listen(3000, () =>
  console.log("Server running on port 3000")
)

MongoDB &
Mongoose

MongoDB — NoSQL Database

Traditional SQL databases store data in rigid tables with fixed columns. MongoDB stores data as JSON-like documents — flexible, nested, and schema-free by default.

SQLMongoDB
TableCollection
RowDocument
ColumnField
Fixed schemaFlexible schema
JOINsEmbedded docs
Mongoose adds a schema and validation layer on top of MongoDB. Without Mongoose, there's nothing stopping someone from saving garbage data in your database.

Schema + CRUD

Student.js 
// 1. Connect
mongoose.connect("mongodb://127.0.0.1/school")

// 2. Schema — defines shape + validation
const studentSchema = new mongoose.Schema({
  name:     { type: String,  required: true },
  age:      { type: Number,  min: 18 },
  isActive: { type: Boolean, default: true }
})

// 3. Model — our interface to the collection
const Student = mongoose.model("Student", studentSchema)

CRUD Operations

controller.js 
// CREATE
await Student.create(req.body)

// READ all
await Student.find()

// READ one by ID
await Student.findById(id)

// UPDATE — { new: true } returns updated doc
await Student.findByIdAndUpdate(
  id, data, { new: true }
)

// DELETE
await Student.findByIdAndDelete(id)
